# UI 设计规范

这份文档是给 AI 和开发者看的页面生成约束。目标不是发散设计，而是让每次新增页面都落在同一套 Ant Design Vue 风格、组件语义和页面风格里，减少随机发挥，保证基础质量。

## 1. 总规则

生成或修改任何 UI 前，必须先遵守以下规则：

1. **优先使用 `ant-design-vue` 组件**：不要重复实现 Button、Table、Form、Modal、Drawer、Tabs、Select、DatePicker、Pagination 等基础组件。
2. **主色使用项目变量**：UnoCSS 使用 `primary` 颜色变量，Less 使用 `@primary` 变量，不在页面里硬编码主色。
3. **样式只能微调，不能另起体系**：页面级样式只处理布局、间距、局部对齐，不重写组件库视觉语言。

## 2. 主题与颜色变量

项目主色已在 UnoCSS 和 Less 中定义，写样式时必须优先使用项目变量。

### 2.1 必须使用的基础值

| 项 | 值 |
| --- | --- |
| 主色变量 | UnoCSS：`primary`；Less：`@primary` |
| 基础字号 `fontSize` | `14` |
| 默认圆角 `borderRadius` | `6` |
| 基础单位 `sizeUnit` | `4` |
| 控件高度 `controlHeight` | `32` |
| 页面背景 | `#f5f5f5` / `#f5f7fa` |
| 容器背景 | `#ffffff` |

主色使用规则：

- UnoCSS 样式优先使用 `text-primary`、`bg-primary`、`border-primary` 等项目变量能力。
- Less 样式优先使用 `@primary`。
- 禁止在页面样式中散落硬编码主色。

### 2.2 颜色使用

| 语义 | 用法 |
| --- | --- |
| Primary | 主按钮、链接、选中态、当前导航、关键数字中的少量强调 |
| Success | 正常、启用、已完成、成功反馈 |
| Warning | 待处理、预警、部分异常 |
| Error | 失败、停用、删除、危险操作 |
| Text Primary | 标题、正文、重要字段 |
| Text Secondary | 辅助说明、时间、备注 |
| Border / Split | 卡片边框、表格分割、区域分隔 |

禁止：

- 禁止同一页面混用多套主色。
- 禁止用大面积主色背景提升“高级感”。
- 禁止使用大面积深色块；页面应以浅色背景和白色容器为主。
- 禁止使用深色侧边栏；`Layout.Sider`、导航菜单、页面主容器不得使用深色背景。
- 组件库禁止使用 dark theme，`ant-design-vue` 组件应保持浅色主题。
- 禁止无语义地使用红、绿、橙。
- 禁止正文低对比度，正文与背景对比度应满足 WCAG AA 的 4.5:1。

## 3. 排版与密度

### 3.1 字号层级

| 层级 | 字号 | 字重 | 场景 |
| --- | --- | --- | --- |
| 页面标题 | 20px / 24px | 600 | 页面主标题 |
| 区块标题 | 16px | 600 | 卡片标题、表单分组标题 |
| 正文 | 14px | 400 / 500 | 表格、表单、普通正文 |
| 辅助说明 | 12px / 13px | 400 | 时间、备注、提示 |
| 关键数字 | 24px / 32px | 600 / 700 | 核心指标 |

数字、金额、编号、百分比使用等宽数字：

```css
font-variant-numeric: tabular-nums;
```

### 3.2 间距

只使用 4px 节奏：

| 名称 | 值 | 用途 |
| --- | --- | --- |
| xs | 4px | 图标与文字 |
| sm | 8px | 按钮组、紧凑元素 |
| md | 12px | 行内分组 |
| lg | 16px | 常规容器内边距 |
| xl | 24px | 页面区块间距 |
| xxl | 32px | 大区块上下留白 |

硬规则：

- 页面外层 padding 默认 `24px`。
- 卡片内边距默认 `16px` 或 `24px`，同页保持一致。
- 同级区块间距默认 `16px` 或 `24px`，不要随机 margin。
- 保持高信息密度：减少无意义留白、过大卡片、过松行高和过大字号，在保证可读性的前提下展示更多有效信息。

## 4. 组件语义

| 场景 | 优先组件 | 约束 |
| --- | --- | --- |
| 页面布局 | `Layout` | 组织整体结构 |
| 层级位置 | `Breadcrumb` | 层级超过两级时使用 |
| 间距 | `Space` | 按钮组、标签组、行内元素 |
| 数据列表 | `Table` | 大量结构化数据、排序、筛选、分页 |
| 数据录入 | `Form` | 字段、校验、提交、回显 |
| 分组容器 | `Card` | 承载一组完整信息，不滥用 |
| 统计数字 | `Statistic` | 突出关键数字 |
| 只读信息 | `Descriptions` | 详情字段展示 |
| 状态标记 | `Tag` / `Badge` | 状态、分类、通知数量 |
| 图表 | `/src/components/EchartsCom.vue` | 基于 echarts 的统一图表组件 |
| 空状态 | `Empty` | 无数据、无权限、未配置 |
| 首屏加载 | `Skeleton` | 首次加载较慢或图文较多 |
| 轻量反馈 | `message` | 成功、警告、错误短反馈 |
| 页面提示 | `Alert` | 静态、可关闭、需要保留的提示 |
| 复杂结果 | `Result` | 重要操作结果或页面级反馈 |
| 二次确认 | `Modal.confirm` | 删除、停用、清空等危险操作 |
| 临时面板 | `Drawer` | 预览、附加任务、子对象创建 |

组件硬规则：

- 不要把所有模块都包成 `Card`；只有一组完整信息才需要卡片。
- 每个操作区最多一个 primary 按钮。
- 危险操作必须二次确认。

## 5. 视觉质量规则

页面是否好看，优先由以下顺序决定：

1. **对齐**：标题、筛选项、按钮、表格列必须落在统一网格上。
2. **层级**：页面标题、区块标题、正文、说明、弱信息必须有明确差异。
3. **密度**：保持高信息密度，不用过大留白、过松行高或过大卡片制造空洞感。
4. **主次**：主按钮唯一；次操作降级；危险操作不放主路径。
5. **细节**：图标、渐变、色块和阴影必须统一风格，不能破坏页面层级。
6. **语义**：背景、图标、状态色和装饰必须来自业务含义，不能随机添加。

页面类型决定视觉强度：

| 页面类型 | 视觉规则 |
| --- | --- |
| 总览 / 仪表盘 | 可使用更强的背景层、数据装饰和图表氛围 |
| 列表 / 表格 | 保持克制，用轻背景、清晰分区和状态标识 |
| 详情页 | 突出主体信息、档案感和信息层级 |
| 表单页 | 优先可读性和操作引导，少装饰 |
| 流程页 | 使用步骤线、时间线或进度关系表达流程 |
| 空状态 / 引导页 | 可使用插画和更强视觉表达 |

装饰语义示例：数据页可用网格、趋势线、数据点；风险页可用低饱和警示色；客户或组织页可用档案卡、关系感；权限或安全页可用审计、锁、盾等元素。

页面不能只是白底上的组件堆叠，应有承载组件的视觉环境。

视觉环境至少包含：

- **背景层**：浅色渐变、低透明网格、细纹理或柔和光斑。
- **内容层**：白色或浅色容器、细边框、轻阴影，形成稳定承载区。
- **重点层**：关键数据、主操作、当前状态有明确强调。
- **装饰层**：SVG 线条、数据点、轨道、低饱和色块等业务语义装饰。

环境强度按页面类型控制：总览、仪表盘、空状态可以更强；列表、表格、表单必须克制。表格、表单和关键数字不能被装饰干扰。

默认视觉配方：

| 项 | 默认做法 |
| --- | --- |
| 页面背景 | 浅灰背景承托白色内容区，不使用大面积深色背景 |
| 内容容器 | 白底 + 细边框，必要时轻阴影 |
| 圆角 | 2px / 4px / 6px / 8px，不使用大圆角 |
| 阴影 | 只用于悬浮层级，不给普通卡片加重阴影 |
| 视觉识别元素 | Logo、导航菜单图标、用户头像、客户头像、指标卡图标等使用 SVG；大尺寸装饰图使用图片 |
| 渐变 | 可以使用，但只用于局部强调，不做整页大背景 |
| 色块 | 使用低饱和颜色，避免高饱和大色块 |
| 动效 | 150ms - 250ms，用于反馈，不用于装饰 |

## 6. 禁止事项

以下内容生成后必须立即删改：

1. 大面积深色背景或深色内容块。
2. 深色侧边栏、深色导航菜单或深色页面主容器。
3. 组件库 dark theme 或深色主题组件。
4. 大面积渐变、强发光、复杂背景图、强阴影、大面积玻璃拟态。
5. 使用单个汉字、字母、Emoji 或特殊字符作为 Logo、头像、菜单图标、指标卡图标等占位图标。
6. 高饱和大色块或无语义彩色标签。
7. 多个 primary 按钮并列。
8. 用自定义 div 模拟 Table、Form、Modal、Select 等基础组件。
9. 硬编码大量颜色、字号、圆角，绕过项目变量。
10. 使用 `TODO`、`Lorem ipsum`、未说明的假数据。
11. 错误提示只写“失败”“异常”。
12. 图表未使用 `/src/components/EchartsCom.vue`，或没有标题、单位、口径、图例、时间范围。

## 7. 生成前检查

AI 写页面前，先在心里完成以下判断，并按判断生成代码：

```text
这个页面的主任务是什么？
主按钮是哪一个？
哪些地方应该直接使用 ant-design-vue 组件？
哪些样式应该用 UnoCSS `primary` 或 Less `@primary`，而不是硬编码主色？
```

如果无法回答这些问题，不要先写复杂样式，先收敛页面结构。

## 8. 完成后检查

提交前必须检查：

- [ ] 主按钮唯一，危险操作有二次确认。
- [ ] 空值、时间、金额、百分比格式统一。
- [ ] 没有随机 margin、随机颜色、随机圆角。
- [ ] 没有无意义图表、无来源指标、未标注假数据。
- [ ] 没有为了“好看”破坏组件库一致性。
- [ ] 中小屏下表格、筛选区、弹窗仍可用。
